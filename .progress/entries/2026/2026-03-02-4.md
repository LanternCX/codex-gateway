## Title
Switch OAuth login default to callback mode with minimal config defaults

## Background / Issue
Device-only OAuth login required explicit OAuth endpoint configuration and did not match the callback-based login flow expected by users familiar with browser redirect and local callback authentication.

## Actions / Outcome
- Approach 1: Added failing tests for callback login flow and minimal config defaults -> reproduced missing callback capability and strict config constraints.
- Approach 2: Implemented callback mode (PKCE + local callback listener + code exchange) and wired `auth login` default to callback with `--mode device` fallback -> aligned CLI behavior with expected browser-based OAuth flow.
- Final approach: Relaxed OAuth config requirements with Codex defaults and updated bilingual docs/config samples -> reduced setup friction while preserving headless fallback.

## Lessons / Refinements
- Callback-first login is easier for interactive local onboarding; device flow should remain fallback, not default.
- Minimal required config (`downstream_api_key`, `upstream.base_url`) significantly improves first-run experience.
