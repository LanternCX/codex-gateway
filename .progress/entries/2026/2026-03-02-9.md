## Date

## Title
Fix opencode compatibility by removing unsupported max_output_tokens forwarding

## Background / Issue
Requests from opencode failed with `Bad Request: {"detail":"Unsupported parameter: max_output_tokens"}` while Apifox requests without token-limit parameters succeeded.

## Actions / Outcome
- Approach 1: Reproduced failure by sending `/v1/chat/completions` with `max_tokens` and confirmed gateway transformed it into upstream `max_output_tokens`.
- Approach 2: Added a failing regression test to assert Codex upstream payload omits `max_output_tokens`.
- Final approach: Removed `max_output_tokens` forwarding in Codex translation, updated API docs to state `max_tokens` is accepted but ignored in `codex_oauth` mode, and verified requests now return `200`.

## Lessons / Refinements
- Codex backend parameter compatibility differs from OpenAI REST semantics; compatibility translation should prefer omission over speculative mapping.
- Keep regression tests for payload translation contracts to prevent future drift.
