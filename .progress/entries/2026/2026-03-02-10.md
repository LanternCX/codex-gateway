## Background / Issue
The repository had no commits yet, and branch policy required a single baseline commit on `main`, stable follow-up work on `dev`, and keeping device login flow out of mainline until production validation.

## Actions / Outcome
- Approach 1: Added a failing CLI test to require `auth login --mode` removal from mainline behavior.
- Approach 2: Removed mainline device-mode CLI flag and synchronized bilingual docs to callback-only login and language-folder doc layout (`docs/en`, `docs/zh-CN`).
- Final approach: Created root baseline commit on `main`, created `dev`, and created `feat/device-oauth-hold` with a dedicated commit that restores device login mode only on that branch.

## Lessons / Refinements
- Branch-policy requirements are easiest to enforce when user-facing flags are removed from protected branches, even if lower-level implementation helpers remain.
- Splitting hold-branch behavior with explicit tests keeps the mainline contract stable and auditable.
