## Title
Implement Codex OAuth OpenAI-compatible gateway MVP

## Background / Issue
Need a self-hosted gateway that supports interactive CLI OAuth login for upstream authentication while exposing OpenAI-compatible endpoints to downstream clients secured by a fixed API key.

## Actions / Outcome
- Approach 1: Drafted architecture and implementation plans first -> established clear scope (`/v1/models`, `/v1/chat/completions`) and runtime-directory storage model.
- Approach 2: Implemented Go CLI, OAuth device flow, token persistence, and refresh manager with unit tests -> delivered login flow and token lifecycle.
- Final approach: Added HTTP gateway middleware + proxy handlers + upstream client + docs/config templates -> delivered end-to-end runnable project with passing tests.

## Lessons / Refinements
- Keeping runtime artifacts (`config.yaml`, `oauth-token.json`) in `--workdir` simplifies server deployment.
- Separating token management (`internal/auth`) from OAuth protocol (`internal/oauth`) keeps refresh logic testable.
- OpenAI-compatible error envelopes make downstream SDK behavior more predictable.
