## Title
Upgrade gateway logging to industrial-grade multi-sink structured telemetry

## Background / Issue
The existing logger only supported a basic stdout sink and minimal request logs, which was insufficient for production observability, secure logging, and request correlation.

## Actions / Outcome
- Approach 1: Added failing tests for configurable sink output (`stdout`/`file`/`both`), request-id propagation, and sensitive value redaction -> confirmed missing production logging capabilities.
- Approach 2: Introduced `internal/logging` enhancements (multi-handler fanout, lumberjack file rotation, redaction hooks, request-id context helpers) and expanded logging config schema with output/file controls.
- Final approach: Wired enhanced logger into CLI/server/upstream paths, propagated `X-Request-ID` across ingress and upstream calls, and synchronized bilingual docs/OpenAPI/config samples with new logging behavior.

## Lessons / Refinements
- Request correlation must be treated as infrastructure (middleware + context + upstream forwarding), not just per-handler logging convenience.
- Industrial logging quality improves quickly when defaults are safe (stdout), file mode is optional, and secret redaction is centralized.
