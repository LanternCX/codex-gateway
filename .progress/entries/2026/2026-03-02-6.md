## Title
Enforce runtime-directory-only config and token path policy

## Background / Issue
User required all runtime static files to remain under the program run directory. Debugging output created confusion around `/tmp` usage, and CLI previously allowed config paths outside `--workdir`.

## Actions / Outcome
- Approach 1: Added failing tests for runtime path resolution and path traversal/outside-workdir rejection -> captured desired runtime policy.
- Approach 2: Implemented runtime path guard in CLI (`resolveRuntimePaths`) and applied it to both `auth login` and `serve` -> config/token paths are now constrained to `--workdir`.
- Final approach: Updated CLI help text and bilingual docs to document the runtime path policy -> behavior and docs now aligned.

## Lessons / Refinements
- Explicit runtime path guards are more reliable than convention-only docs.
- Clarifying debug-temp output versus application runtime files avoids operational confusion.
