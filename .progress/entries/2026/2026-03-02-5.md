## Title
Debug callback unknown_error and align OAuth authorize request with opencode

## Background / Issue
Browser callback login returned `unknown_error` during authorization. Need to cross-check request parameters against `../opencode` Codex OAuth flow and remove incompatibilities.

## Actions / Outcome
- Approach 1: Reproduced generated authorize URL and compared query params with `opencode` implementation -> found mismatch in `originator` and redirect host defaults.
- Approach 2: Added failing tests for config callback defaults and authorize URL originator -> locked expected compatibility behavior.
- Final approach: Switched defaults to `redirect_host=localhost`, `originator=opencode`, propagated config fields through CLI/oauth client, and updated docs/examples -> callback authorize URL now matches opencode pattern.

## Lessons / Refinements
- For OAuth integrations with non-public client apps, exact query parameter values can be compatibility-critical.
- Diffing generated authorize URL against known-good implementation is the fastest way to isolate browser-side unknown errors.
